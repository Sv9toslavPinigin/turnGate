# TurnGate — ТЗ для дизайнера

Android VPN-клиент, использующий VK TURN-прокси для туннелирования трафика через WireGuard. Приложение запускает нативный Go-бинарник, который устанавливает TURN-соединение с серверами VK, а затем поднимает WireGuard-туннель поверх него. Основные задачи дизайна — сделать интерфейс понятным для пользователей без технического бэкграунда, упростить импорт конфигов, дать адекватный UX для решения капчи VK и просмотра логов диагностики.

---

## Стек и ограничения

- **Платформа:** Android (minSdk 26, targetSdk 35)
- **UI:** Jetpack Compose + Material 3
- **Тема:** только тёмная
- **Ориентация:** только портретная (по факту — проверить)
- **Единая Activity**, кастомная навигация через enum-состояние (5 экранов)

## Что такое "профиль"

Профиль — это набор параметров одного VPN-подключения. У пользователя их может быть несколько, переключается на главном экране. Поля профиля:

- **Name** — произвольное имя (например "Netherlands")
- **TURN Server URL** — ссылка на VK-звонок `https://vk.com/call/join/...`
- **Peer Address** — IP:PORT удалённого сервера (`1.2.3.4:56000`)
- **Listen Address** — локальный адрес, по умолчанию `127.0.0.1:9000`
- **Connections (n)** — число параллельных TURN-соединений (1–16, дефолт 16)
- **WireGuard Config** — целиковый wg-quick конфиг (multiline, моноширинный)
- **Manual Captcha** — флаг, принудительно использовать ручное решение капчи

## Состояния подключения

`ConnectionState`: `DISCONNECTED`, `CONNECTING`, `CONNECTED`, `DISCONNECTING`, `ERROR`.

В UI различаются:
- цветом иконки щита (серый / оранжевый пульс / зелёный с glow / красный)
- цветом и текстом кнопки (Connect blue / Connecting orange со спиннером / Disconnect red)
- текстом статуса под иконкой (свободный, идёт от ViewModel: "Starting proxy...", "Proxy running, starting VPN...", "Connected", "Captcha required...", "Invalid config link" и т.п.)

## Экраны

### 1. Home (главный)

Композиция сверху вниз:

1. **Top bar** — две IconButton по краям:
   - Слева: `+` — открывает модалку импорта (см. ниже). Disabled если подключено.
   - Справа: шестерёнка — переход на Settings.
2. **Заголовок** "TurnGate" — крупно, жирно, linear gradient (Blue → Cyan).
3. **Profile Picker** — кнопка-дропдаун с именем текущего профиля и стрелкой вниз. Disabled если подключено. Открывает список профилей: у каждого пункта — имя и справа маленькая текстовая кнопка "Edit" (переход в редактор). Если профилей нет — не показывается.
4. **Щит** (по центру, основной визуальный элемент):
   - Размер ~120dp
   - DISCONNECTED — серый
   - CONNECTING / DISCONNECTING — оранжевый, пульсация (1.0 ↔ 1.15)
   - CONNECTED — зелёный + blurred glow (40dp blur, alpha 0.4)
   - ERROR — красный
5. **Статус-текст** под щитом — маленький, Secondary color.
6. **Лог-панель** (если есть логи прокси): высота 120dp, тёмный фон `#0A0F1A`, моноширинный шрифт 10sp, цвет строк `#7DD3FC`, auto-scroll к последней строке. Показывается только во время подключения / после попытки.
7. **Connect button** — Full-width, 60dp высоты, скруглён 16dp, shadow 12dp. Цвет — Blue/Orange/Red по состоянию. При подключении — спиннер слева и текст "Connecting...".

**Модалка импорта профиля** (AlertDialog):
- Заголовок "Add Profile"
- Кнопка "Paste from Clipboard" (primary) — пытается распарсить из буфера turnbridge://... ссылку и создать профиль.
- Кнопка "Add Manually" (outlined) — создаёт пустой профиль и открывает редактор.
- "Cancel" снизу.

## Captcha WebView (диалог поверх Home)

Появляется автоматически, когда прокси не смог решить капчу автоматически (или включён Manual mode). Сейчас — Dialog на 90% высоты экрана:

- Внутри карточка Surface, скругление 16dp, padding 12dp
- **Шапка**: слева крупно "Captcha #N" (если N > 0) или "Solve Captcha", под ним — маленьким Secondary текст "Multiple captchas may follow in sequence". Справа — кнопки `Refresh` (outlined) и `Cancel` (красная).
- **WebView** — занимает всё свободное место, белый фон, скруглён 8dp. Внутри загружается страница с локального HTTP-сервера, который запустил прокси (обычно VK captcha_not_robot или слайдер).

Диалог закрывается:
- автоматически, когда прокси сигналит что токен получен
- автоматически, когда страница WebView показывает "Done!" / "close the page"
- автоматически, когда сервер прокси закрылся (ERR_CONNECTION_REFUSED в main frame)
- по кнопке Cancel — с полным отключением VPN

Dismiss на тап вне диалога и на back-нажатие **отключены** (чтобы пользователь не свернул случайно).

> Важно для дизайна: капч может быть несколько подряд. Нужна ненавязчивая индикация прогресса / счётчика "Captcha #1 → Captcha #2 → подключено".

## Навигация и deep links

- Навигация примитивная: enum-стейт Screen, переходы через setter.
- Back на любом экране кроме Home — возвращает на предыдущий; на Home — закрывает приложение (штатно).
- Deep link `turnbridge://<base64-json>` принимается как VIEW-intent, профиль создаётся автоматически и выбирается. Можно прийти из буфера обмена или по клику на ссылку.

## Что хотелось бы от дизайна

1. **Главный экран**: сделать "вау"-момент для щита/подключения, но без визуального шума. Сейчас — минималистичный OG-look тёмной VPN-ки.
2. **Профили**: выбор профиля должен быть быстрее. Возможно — горизонтальный карусель вверху вместо дропдауна, с большими тапабельными карточками.
3. **Импорт**: сделать процесс "вставил → получил готовый профиль" более очевидным. Возможно, preview импортируемого профиля перед сохранением.
4. **Captcha-диалог**: подумать про sequential captchas (их может быть 3-5 подряд) — чтобы пользователь не думал что приложение сломалось.
5. **Логи**: сейчас это полу-дебаг-экран. Для обычного юзера — добавить "дружелюбный" режим с простыми сообщениями ("Connecting... Authenticated... Captcha solved... Connected") вместо сырого stdout.
6. **Empty states**: нет профилей, нет логов, первый запуск — сейчас пустовато.
7. **Settings**: можно наполнить реальным контентом (LAN bypass, per-app VPN, kill-switch, авто-старт).
8. **Иконка приложения**: щит — ок, но просится фирменный знак.

## Фичи под капотом (для контекста)

- Нативный бинарник Go (`libvkturnproxy.so`) запускается как процесс, stdout парсится на события капчи
- При событии `ManualCaptchaRequired` показывается WebView диалог, localhost:8765 отдаёт captcha-страницу от прокси
- Логи пишутся и в память (StateFlow), и в файл `vpn_tunnel.log` с ротацией при >500KB (оставляет последние 60%)
- Профили хранятся в SharedPreferences как JSON-массив; есть автомиграция из старого формата
- WireGuard-туннель поднимается через `com.wireguard.android:tunnel` поверх локального прокси на 127.0.0.1:9000
